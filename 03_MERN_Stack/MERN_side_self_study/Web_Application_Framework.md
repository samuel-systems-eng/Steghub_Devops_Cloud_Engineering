# MERN SELF STUDY

## Question:

Familiarize with concept of web application framework. Know server-side (backend) and a client side (front-end) frameworks and what they are used for

### Answer:

A web framework is a pre-built library of tools and rules that simplifies building web applications.

**Server-Side (Backend) Frameworks**

    • Definition: Software running on the web server (e.g., Express.js, Django).

    • Purpose: Handles database operations, user authentication, API endpoints and business logic.

#### Examples:

* Express.js: A minimal, unopinionated JavaScript framework for Node.js (the "E" in MERN).
  
* NestJS: A heavily structured, TypeScript-first Node.js framework optimized for enterprise scale.
* Django: A high-level, batteries-included Python framework with built-in security features.
* Spring Boot: A robust, production-ready framework built for the Java ecosystem.
* Laravel: A feature-rich PHP framework known for elegant syntax and built-in database migration tools.
  
* ASP.NET Core: A high-performance, open-source framework by Microsoft powered by C#.  

**Client-Side (Frontend) Frameworks**

    • Definition: Software running inside the user's web browser (e.g., React, Angular).

    • Purpose: Manages user interfaces, handles visual rendering, and captures user input.

#### Examples:
  * React: A component-based JavaScript library (the "R" in MERN) favored for building reusable UI pieces.
  
  * Vue.js: A lightweight, beginner-friendly JavaScript framework known for its gradual learning curve.
  * Angular: A robust, TypeScript-based framework built by Google, highly favored by large enterprises.
  * Svelte: A compiler-based framework that shifts work from the browser to the build step for peak performance.
  * Next.js: A framework built on top of React that handles server-side rendering (SSR) for better SEO.
